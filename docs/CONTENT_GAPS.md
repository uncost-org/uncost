# Content gaps — grouped by field

Every `[CONTENT NEEDED]` marker in the built site, grouped by field slug — so one row is one writing task, even when it spans many pages. Author each from the July Movement Plan (`a2b640`), the dossiers, and the per-route copy captures in `docs/EXPORT_COPY_DRAFTS/` (which hold the export's design-authored draft for that field). No field is invented copy; each is a deliberate honest gap.

**122 distinct fields across 210 markers.**

**Flag tiers in the captures (the 190 former "uncited" flags, re-tiered).** In `docs/EXPORT_COPY_DRAFTS/` each factual/numeric claim now carries one of three tiers instead of a flat "uncited", because "not in the July PDF" is not the same as "no source":

- **`⚠ UNCITED` — 136.** No source anywhere. A real gap; author or drop.
- **`⚠ SOURCED ELSEWHERE` — 9.** A real source exists outside the July PDF; the fix is verify-and-**register in `sources/register.csv`**, not delete (e.g. the Home Harvard JCHS / BLS statistics; "Water is next" as founder-ratified canon).
- **`⚠ REPO-VERIFIABLE` — 45.** Checkable against this repository — a stronger receipt than the PDF (e.g. the 10-policy / 7-project / 15-sector counts, which `audit_repository.py` enforces).

**Config-gated block not in the counts above:** `/pledge/thanks/` carries a **founding-signer recognition** block that is present in the template but dark by default (`features.foundingSigners`, off). Its gaps (`thanks-founding-signers`, `thanks-founding-signers-heading`) only render when activated, so they are not in the 210. **It cannot be activated until a public-display consent purpose exists in the UNP-57 consent model and has been collected** — see the template comment and `website/src/_data/features.js`.
