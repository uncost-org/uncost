# Security policy

## Reporting

Please do not open a public issue for a vulnerability, exposed credential, personal-data leak, or safeguarding concern. Use GitHub's private vulnerability reporting / Security Advisory flow for this repository.

Include the affected path or component, reproduction details, likely impact, and any safe mitigation you have identified. Do not access or retain more data than necessary to demonstrate the issue.

## Scope

This repository currently contains public documentation, draft project material, and a placeholder website scaffold. Access-gated preview deployments of pull-request builds are founder-authorized (2026-07-19): previews must be served behind Cloudflare Access, are deployed with a robots-disallow and noindex posture, carry no forms or data collection, and serve no production domain. Public launch, production deployment, DNS changes, data systems, pledge collection, donations, and community systems remain not authorized by this repository state.

Security reports do not authorize testing against production services, third parties, or personal data.
